1. ARCHITECTURE AT A GLANCE
================================================================================

MaxHeap (Data)
  - insert(), delete(), buildHeap(), heapSort()
  - Records every compare/swap step
  - Tracks comparisons, swaps
  - Stores [type, data, stats, msg]

HeapVisualizer (Canvas)
  - drawTree() - Renders tree structure (BFS layout)
  - drawArray() - Bar chart visualization
  - highlight() - Color-coded state display

HeapSortApp (Controller)
  - playAnimation() - Plays recorded steps
  - renderStep() - Interprets step type
  - stepForward/Backward() - Manual navigation
  - updateDisplay() - Updates UI

Flow: Algorithm (record steps) → Playback (from array) → Render (canvas+DOM)

================================================================================
2. KEY INSIGHT: ANIMATION RECORDING
================================================================================

WRONG: Algorithm → Real-time animation
  Problem: Hard to control speed, can't step backward

CORRECT: Algorithm (record steps) → Playback (from array)
  Benefit: Perfect replay, speed control, step backward

Implementation:
  1. During algorithm: addAnimationStep(type, {indices, data, msg})
  2. Store steps: this.animationSteps = [...]
  3. On playback: renderStep(animationSteps[currentStep])
  4. User controls: speed slider, pause, step forward/back

================================================================================
3. ANIMATION STEP TYPES
================================================================================

compare         - Compare two indices (highlight RED)
swap            - Swap two indices (highlight YELLOW)
insert          - Add to heap (clear highlights)
delete          - Remove from heap (clear highlights)
heapify         - Heapify operation (highlight single index)
build_complete  - Build heap finished (clear highlights)
extract         - Element moved to sorted array
sort_complete   - Sort finished (highlight all GREEN)

Each step: {type, indices, heap_state, comparisons, swaps, message}

4. VISUALIZATION PATTERNS
================================================================================

TREE LAYOUT (BFS Algorithm):
  - Levels: Math.ceil(Math.log2(size + 1))
  - Queue: [{index: 0, level: 0, left: 0, right: width}]
  - Position: x = (left + right) / 2, y = startY + level * levelHeight
  - Subtrees: left [left, x), right [x, right)

BAR CHART:
  - Height: value * (height - 80) / max_value
  - Width: (width - 40) / length - 10
  - Grid: Horizontal reference lines

COLORS:
  #6366f1 (Indigo)  = Default state
  #f87171 (Red)     = Comparing (active)
  #fbbf24 (Amber)   = Swapping, Root
  #34d399 (Green)   = Sorted, Completed
  #0f172a (Dark)    = Background

EFFECTS:
  - shadowBlur: 15 (node halos)
  - shadowColor: match element color
  - Reset: shadowBlur = 0 after draw

================================================================================
5. PLAYBACK LOOP
================================================================================

playAnimation()
  → Set animationSteps = heap.animationSteps
  → Set currentStep = 0, isAnimating = true
  → Call playNextStep()

playNextStep()
  → Get step = animationSteps[currentStep]
  → Call renderStep(step)
  → delay = Math.max(100, (11 - speed) * 100)
  → setTimeout(() => {currentStep++; playNextStep()}, delay)

renderStep(step)
  → Switch on step.type
  → visualizer.highlight(indices)
  → Update stats (comparisons, swaps, size)
  → visualizer.draw()

Controls:
  - Play/Pause: Toggle isAnimating
  - Step Forward: currentStep++
  - Step Backward: currentStep--
  - Speed: 1-10 scale (1000-100ms delay)

9. PERFORMANCE TIPS
================================================================================

Canvas:
  - Clear once per frame: ctx.fillRect(0, 0, width, height)
  - Pre-calculate positions (BFS before drawing)
  - Set shadowBlur = 0 after use
  - Selectively apply shadows (not every element)

Animation:
  - Minimum delay 100ms per step
  - Speed 1-10 maps to 1000-100ms
  - Store steps once (don't recreate)

Responsive:
  - Listen to window resize
  - Recalculate canvas dimensions dynamically
  - Use clientWidth/clientHeight
  - Call draw() after resize

Event Listener with Validation:
  insertBtn.addEventListener('click', () => {
      const value = parseInt(insertInput.value, 10);
      if (isNaN(value)) {
          insertInput.classList.add('input-error');
          setTimeout(() => insertInput.classList.remove('input-error'), 600);
          return;
      }
      this.insert(value);
  });
