# Step 63 - 최적화 스크린샷 검증

## 실행 내용

step62 최적화 구현 완료 후, 최적화로 인해 UI가 깨지지 않았는지 스크린샷으로 검증한다.

**필요한 파일:**

- step027_설계_chunk*.md (레이아웃 설계)

**이 단계에서 절대로 plan mode를 사용하지 않는다.**

**스크린샷 없이 통과 처리 금지. 반드시 Claude가 직접 스크린샷을 눈으로 확인한다.**

## 검증 절차

1. step027_설계_chunk*.md에서 레이아웃 구조, 배치, 비율, 반응형 브레이크포인트 등 설계 명세를 추출한다.
2. 개발 서버를 실행하고 Playwright로 주요 뷰포트별 스크린샷을 촬영한다.
3. Claude가 스크린샷을 직접 Read하여 설계 명세와 대조한다.
4. **문제 발견 시:**
   - 최적화로 인해 깨진 부분을 구체적으로 기록
   - 해당 소스 코드 수정
   - 스크린샷 재촬영
   - Claude가 다시 직접 확인
   - **설계와 일치할 때까지 반복**
5. 모든 뷰포트에서 레이아웃이 설계와 일치하면 통과

## 검증 축 생성 규칙

**사전에 고정된 검증 항목 목록을 사용하지 않는다.**

1. step027_설계_chunk*.md의 설계 명세에서 검증해야 할 시각적 요소를 식별한다
2. 식별된 요소마다 하나의 검증 축을 생성한다
3. 설계에 명시되지 않은 항목을 임의로 추가하지 않는다

## 디자인 자연스러움 검토

설계 명세 일치 여부와 별개로, 스크린샷을 보고 **디자인적으로 어색한 부분**이 없는지 추가 검토한다.

**최대한 섬세하게 파악한다. 사소한 위화감도 놓치지 않는다.**

**검토 축 생성 규칙:**

- 사전에 고정된 검토 항목 목록을 사용하지 않는다
- 스크린샷에서 실제로 보이는 시각 요소를 하나씩 식별한다
- 식별된 요소마다 "이 요소가 시각적으로 자연스러운가?"를 판단하는 검토 축을 생성한다
- 스크린샷에 없는 요소에 대한 검토 축을 임의로 추가하지 않는다

**문제 발견 시:**

1. 어색한 부분을 구체적으로 기록 (어떤 요소가 왜 어색한지)
2. 해당 소스 코드(HTML/CSS/JS) 수정
3. 스크린샷 재촬영
4. Claude가 다시 직접 확인
5. **자연스럽고 어색하지 않은 디자인이 될 때까지 반복**

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용한다.

서브에이전트는 항상 haiku를 사용한다.
