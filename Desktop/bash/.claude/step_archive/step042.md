# Step 42 - 테스트 실행: c8 커버리지 검증

작성된 모든 테스트를 실행하여 통과 여부를 확인한다.

## 실행 내용

테스트 실행 시 반드시 c8을 함께 실행하여 커버리지를 측정한다:

```bash
npx c8 --check-coverage --lines 80 --branches 70 --functions 80 vitest run
```

- `--check-coverage`: 커버리지 목표 미달 시 자동 실패 처리
- 커버리지 리포트는 `step046_테스트결과_chunk*.md`에 포함하여 기록한다
- 커버리지 목표 미달 파일이 있으면 Step 44로 돌아가 테스트를 보완한다

실패한 테스트는 원인을 분석하고 기록한다.

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용하여 테스트를 실행한다.

**테스트 실행 단계에서 절대로 plan mode를 사용하지 않는다.**

**테스트 결과는 청크 단위로 저장한다:**

```
step046_테스트결과_chunk1.md (500줄 이하)
step046_테스트결과_chunk2.md (500줄 이하)
step046_테스트결과_chunk3.md (500줄 이하)
...
```

**작성 규칙**:

- 각 청크는 500줄 이하로 작성 (성능 최적화)
- `.claude/hooks/research-validator.ps1`에서 각 청크 검증 (BOM/CRLF/줄수/파일크기)
- 청크 그대로 유지 (병합 안 함)

서브에이전트는 항상 haiku를 사용한다.
