# Step 84 - 통합 테스트: c8 커버리지 갭 분석

프로젝트 아키텍처와 현재 테스트 현황을 분석하여 최적의 테스트 비율을 동적으로 결정한다.

## 실행 내용

`npx c8 report --reporter=json` 으로 현재 단위 테스트 커버리지를 분석하여, 커버리지가 낮은 모듈 간 연결 지점을 통합 테스트 대상으로 우선 선정한다. 단위 테스트로 커버되지 않는 모듈 간 인터페이스를 집중적으로 테스트한다.

합리적인 선에서 최대한 많은 서브에이전트를 병렬로 사용한다.

**통합 테스트 단계에서 절대로 plan mode를 사용하지 않는다.**

서브에이전트는 항상 haiku를 사용한다.
